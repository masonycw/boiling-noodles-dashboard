# 滾麵專案 - 系統架構升級藍圖與檢核表

**建立日期：** 2026-03-05
**目標：** 解決 Streamlit 記憶體滿載與讀取緩慢問題，引進 PostgreSQL 作為資料中轉站。

---

## 🚀 Phase 1: 基礎工程與中轉站建置 (當下核心任務)
- [ ] **Step 1: 設定 GCP 主機 2GB Swap 虛擬記憶體** (防止主機當機)
- [ ] **Step 2: 在 GCP 主機上安裝 PostgreSQL**
- [ ] **Step 3: 調整 PostgreSQL 記憶體設定檔** (降低 `shared_buffers` 適應 2GB RAM 環境)
- [ ] **Step 4: 建立資料庫與連線專屬帳號**
- [ ] **Step 5: 設計核心資料表 Schema**
    - `orders_fact` (所有明細)
    - `daily_revenue_agg` (每日營收表)
    - `member_profile_agg` (會員輪廓表)
- [ ] **Step 6: 撰寫 `data_pipeline.py` (ETL) 腳本** 處理歷史資料的首次清洗與匯入
- [ ] **Step 7: 撰寫增量更新邏輯** (設定每日排程 Cronjob 觸發)

## ⚡️ Phase 2: Streamlit 效能解放與平滑轉移
- [ ] **Step 1: 建立 Streamlit 測試頁面**，用 SQL 連接 PostgreSQL 撈取資料
- [ ] **Step 2: 驗證數據**：比對新舊資料庫算出的「每日營業總額」是否 100% 吻合
- [ ] **Step 3: 驗證數據**：比對新舊資料庫算出的「會員客單價及頻率」是否無誤
- [ ] **Step 4: 替換營收圖表** (`views/operational.py`) 為 SQL 讀取版
- [ ] **Step 5: 替換會員圖表** (`views/member.py`) 為 SQL 讀取版
- [ ] **Step 6: 替換預測圖表** (`views/prediction.py`) 為 SQL 讀取版
- [ ] **Step 7: 移除舊包袱**：拔除舊有耗時的 Pandas 在記憶體中的 Cache 機制
